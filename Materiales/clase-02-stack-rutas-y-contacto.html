1. Objetivos de la clase

En la primera clase vimos cómo preparar el entorno y crear un primer Hola Mundo con React y Vite. En esta segunda clase convertimos ese ejemplo inicial en una estructura mínima de aplicación. Todavía no buscamos una app completa, sino una base clara para crecer.

El cambio principal respecto a la clase 1 es conceptual. Ya no pensamos en una única pantalla dentro de App.jsx. Empezamos a pensar en una aplicación formada por piezas: una página de inicio, una página de contacto, una navegación común y un archivo donde se declaran las rutas. Esa separación es pequeña, pero prepara al alumnado para proyectos más reales.

Al terminar
  • Entenderemos que una app React puede tener varias URLs internas.
  • Instalaremos la dependencia adicional react-router-dom.
  • Separaremos el código en components/, pages/ y routes/.
  • Crearemos una ruta de inicio / y una ruta de contacto /contacto.
  • Usaremos Link para navegar sin recargar toda la página.
  • Dejaremos una carpeta Backend/ preparada, sin desarrollar Express todavía.
  • Relacionaremos cada archivo con el flujo real de la aplicación.
  • Repasaremos la sintaxis JavaScript y JSX que aparece en el código.
Alcance de clase 2

No vamos a crear todavía formularios funcionales, estado con useState, peticiones HTTP, Express ni bases de datos. Esta clase trata de estructura, navegación y lectura del flujo.

2. Orden de trabajo

Para evitar líos, la clase 2 debe seguir siempre el mismo orden. No empezamos creando rutas si antes no hemos instalado la dependencia, y no creamos enlaces si antes no tenemos claras las páginas.

Este orden también ayuda a explicar causa y efecto. Primero creamos las páginas porque son el contenido que queremos mostrar. Después creamos las rutas porque necesitamos decir qué URL muestra cada página. Luego creamos la navegación porque los enlaces necesitan apuntar a rutas que ya existen. Por último, conectamos todo en App.jsx.

Secuencia recomendada
  1. Partir del proyecto React creado en la clase 1.
  2. Abrir la raíz del proyecto en VS Code.
  3. Comprobar que estamos en la carpeta correcta desde la terminal.
  4. Instalar dependencias si el proyecto acaba de descargarse.
  5. Instalar la dependencia nueva de rutas: react-router-dom.
  6. Crear las carpetas components/, pages/ y routes/.
  7. Crear componentes pequeños: Texto.jsx y Boton.jsx.
  8. Crear después las páginas: Inicio.jsx y Contacto.jsx.
  9. Crear después AppRoutes.jsx para asociar URLs con páginas.
  10. Crear Navegacion.jsx con enlaces internos mediante Link.
  11. Actualizar App.jsx para montar BrowserRouter, navegación y rutas.
  12. Preparar la carpeta Backend/ sin desarrollar Express todavía.
  13. Probar el frontend con npm run dev.
  14. Verificar con npm run lint y npm run build.

2.1 Comandos en orden

Si el proyecto ya está creado y estamos empezando la clase 2, estos son los comandos principales: 

npm install
npm install react-router-dom
npm run dev

Si queremos comprobar que todo compila al final: 

npm run lint
npm run build

En clase conviene escribir los comandos en la terminal integrada de VS Code y detenerse después de cada uno. Si el comando falla, no se debe seguir al siguiente paso. Por ejemplo, si npm install react-router-dom falla, los imports de BrowserRouter, Routes, Route o Link no funcionarán.

3. Qué vamos a construir

El objetivo práctico es tener un stack muy pequeño con dos pantallas: una página de inicio y una página de contacto. El backend solo queda preparado como carpeta para que se vea la arquitectura global.

Este stack no tiene todavía datos dinámicos, formularios, estado ni llamadas a una API. Eso es intencionado. La clase se centra en que el alumnado entienda qué archivo cumple cada función y cómo se conectan las piezas. Una vez entendido esto, será mucho más fácil añadir interactividad.

URL Archivo de página Qué muestra
/ src/pages/Inicio.jsx Datos básicos del curso y bloque de conceptos.
/contacto src/pages/Contacto.jsx Página interna de contacto con texto explicativo.

Para navegar entre esas páginas usamos un componente llamado Navegacion. Para decidir qué página se muestra según la URL usamos AppRoutes.

Ejemplo mental

Podemos imaginar AppRoutes.jsx como una tabla de equivalencias: "si la URL es esta, muestra esta página". La navegación no decide qué se muestra; solo cambia la URL. La decisión la toman las rutas.

En package.json aparecen dos bloques habituales: dependencies y devDependencies. Para el alumnado principiante, la diferencia se puede explicar así:

  • dependencies: paquetes que la aplicación necesita para funcionar.
  • devDependencies: herramientas necesarias durante el desarrollo.

React y React Router son dependencias de la aplicación. Vite y ESLint son herramientas de desarrollo.

5. Arquitectura del stack

La raíz del proyecto contiene el frontend con React y Vite. La carpeta Backend/ queda reservada para Node y Express. Aunque estén en la misma carpeta de trabajo, frontend y backend son responsabilidades diferentes.

Una regla útil para decidir dónde colocar un archivo es preguntarse: "¿este archivo representa una página completa, una pieza reutilizable o una configuración de rutas?". Si es una pantalla completa, irá a pages/. Si es una pieza que puede aparecer en varias páginas, irá a components/. Si decide qué URL muestra qué página, irá a routes/.

6. Flujo completo de la app

Este es el recorrido más importante de la clase. Conviene explicarlo despacio y volver a él varias veces.

La clave es distinguir entre "cargar la aplicación" y "navegar dentro de la aplicación". La primera carga ocurre cuando abrimos localhost:5173. A partir de ahí, React Router puede cambiar entre páginas internas sin volver a cargar todo desde cero.

1. Vite
Sirve index.html en el navegador.
2. index.html
Tiene div#root y carga main.jsx.
3. main.jsx
Busca root y renderiza App.
4. App.jsx
Activa BrowserRouter.
5. Navegacion.jsx
Muestra enlaces con Link.
6. AppRoutes.jsx
Relaciona URL con página.

6.1 Flujo al abrir la home

  1. El navegador abre http://localhost:5173/.
  2. Vite entrega index.html.
  3. index.html carga /src/main.jsx.
  4. main.jsx monta App en div#root.
  5. App activa BrowserRouter.
  6. AppRoutes ve que la URL es /.
  7. React muestra Inicio.

6.2 Flujo al pulsar Contacto

  1. El usuario pulsa el enlace Contacto.
  2. Link cambia la URL a /contacto.
  3. BrowserRouter informa a React Router de la ruta actual.
  4. AppRoutes encuentra path="/contacto".
  5. React muestra el componente Contacto.
Frase clave

App.jsx no pregunta manualmente qué URL hay. AppRoutes.jsx declara las rutas y React Router decide qué página debe renderizar.

7.5 src/main.jsx

// Importamos StrictMode desde React.
// StrictMode ayuda durante el desarrollo avisando de posibles problemas.
// En este curso lo usamos solo aqui, en el punto de entrada de la aplicacion.
import { StrictMode } from 'react'

// Importamos createRoot desde react-dom/client.
// Sirve para decirle a React en que elemento del HTML debe montar la aplicacion.
// Tambien se usa una vez, en main.jsx.
import { createRoot } from 'react-dom/client'

// Importamos los estilos globales.
// Este CSS afecta a toda la aplicacion, por eso se importa en main.jsx.
import './index.css'

// Importamos el componente principal App.
// A partir de App iremos componiendo el resto de la interfaz.
import App from './App.jsx'

// El navegador carga index.html.
// Dentro de index.html existe un div vacio con id="root".
// React usara ese div como contenedor principal de toda la aplicacion.
const contenedor = document.getElementById('root')

// createRoot crea la conexion entre React y el div del HTML.
// render muestra el componente App dentro de ese contenedor.
createRoot(contenedor).render(
  <StrictMode>
    <App />
  </StrictMode>,
)

7.6 src/App.jsx

// Importamos BrowserRouter desde React Router.
// BrowserRouter activa el sistema de rutas usando la URL del navegador.
// Debe envolver a los componentes que usan rutas o enlaces internos.
import { BrowserRouter } from 'react-router-dom'

// Importamos los estilos propios de App.
// Este archivo se importa aqui porque sus clases se usan en la estructura general.
import './App.css'

// Importamos la navegacion principal.
// La navegacion se separa en components/ porque no es una pagina completa.
import Navegacion from './components/Navegacion.jsx'

// Importamos la configuracion de rutas.
// AppRoutes vive en routes/ y decide que pagina se muestra segun la URL.
import AppRoutes from './routes/AppRoutes.jsx'

// App es el componente principal de la aplicacion.
// Para clase 2 no metemos aqui el contenido de Inicio ni Contacto.
// App solo monta la estructura comun: router, navegacion y rutas.
function App() {
  return (
    <BrowserRouter>
      <main className="pagina">
        <Navegacion />
        <AppRoutes />
      </main>
    </BrowserRouter>
  )
}

export default App

7.10 src/routes/AppRoutes.jsx

// Importamos Routes y Route desde React Router.
// Routes agrupa todas las rutas disponibles.
// Route relaciona una URL con el componente que debe mostrarse.
import { Route, Routes } from 'react-router-dom'

// Importamos las paginas que queremos asociar a una URL.
// Cada pagina vive en su propio archivo para que el proyecto pueda crecer.
import Contacto from '../pages/Contacto.jsx'
import Inicio from '../pages/Inicio.jsx'

// Este componente centraliza las URLs de la aplicacion.
// Para anadir una pagina nueva, normalmente anadiremos aqui una nueva Route.
// Asi evitamos llenar App.jsx con condicionales o contenidos de todas las paginas.
function AppRoutes() {
  return (
    <Routes>
      <Route path="/" element={<Inicio />} />
      <Route path="/contacto" element={<Contacto />} />
    </Routes>
  )
}

export default AppRoutes

9. JavaScript y JSX usados en este stack

Esta sección conecta el código con la teoría de JavaScript necesaria. No se trata de memorizarlo todo, sino de reconocer las piezas que ya aparecen en el proyecto.

9.1 Imports y exports

Un proyecto React se divide en archivos. Para usar algo definido en otro archivo se importa. Para permitir que otros archivos lo usen, se exporta.

Esta idea es fundamental: cada archivo puede tener una responsabilidad pequeña. Por ejemplo, Contacto.jsx solo se preocupa de la página Contacto. Si otro archivo necesita mostrar esa página, la importa. Así evitamos tener todo el proyecto dentro de un único archivo largo. 

import App from './App.jsx'
export default App
Sintaxis Significado
import App from './App.jsx' Trae el export por defecto del archivo App.jsx.
import { Link } from 'react-router-dom' Trae una exportación con nombre llamada Link.
export default Inicio Permite importar Inicio desde otro archivo.

Ejemplo paso a paso

// 1. En Contacto.jsx creamos el componente.
function Contacto() {
  return <h1>Contacto</h1>
}

// 2. Lo exportamos para que otros archivos puedan usarlo.
export default Contacto

// 3. En AppRoutes.jsx lo importamos.
import Contacto from '../pages/Contacto.jsx'

Si olvidamos el export default Contacto, el archivo AppRoutes.jsx no podrá importarlo correctamente. Si escribimos mal la ruta del import, React tampoco encontrará el archivo.

9.2 Constantes

Usamos const cuando declaramos un valor que no vamos a reasignar. 

const tituloCurso = 'Desarrollo de Aplicaciones con React y Node.js'
const duracionHoras = 30
const numeroSesiones = 6
const horasPorSesion = duracionHoras / numeroSesiones

Aquí aparecen strings, números y una operación matemática. Después esos valores se insertan en JSX con llaves.

En este ejemplo, tituloCurso es una cadena de texto, duracionHoras y numeroSesiones son números, y horasPorSesion es el resultado de una operación. No necesitamos escribir directamente "5"; dejamos que JavaScript lo calcule.

Ejemplo equivalente

const duracionHoras = 30
const numeroSesiones = 6

// JavaScript calcula 30 / 6.
const horasPorSesion = duracionHoras / numeroSesiones

Si más adelante el curso tuviera 40 horas y 8 sesiones, solo habría que cambiar las constantes base. El cálculo seguiría funcionando.

9.3 Funciones como componentes

En React, una forma básica de crear componentes es escribir una función que devuelve JSX. 

function Contacto() {
  return (
    <section>
      <h1>Contacto</h1>
    </section>
  )
}

El nombre del componente se escribe con mayúscula inicial. Por eso usamos Contacto, Inicio o Navegacion.

React distingue entre etiquetas HTML normales y componentes propios. Una etiqueta como <section> empieza en minúscula. Un componente como <Contacto /> empieza en mayúscula. Esta convención ayuda a React a saber qué debe renderizar.

Ejemplo Qué es
<h1> Etiqueta HTML.
<section> Etiqueta HTML.
<Inicio /> Componente React.
<Navegacion /> Componente React.

9.4 JSX

JSX se parece a HTML, pero no es HTML puro. Vive dentro de JavaScript y permite insertar expresiones con llaves. 

<h1>{tituloCurso}</h1>
<p>El curso tiene {duracionHoras} horas.</p>

Todo lo que va entre { } se evalúa como JavaScript. Puede ser una variable, una operación o una expresión sencilla.

Ejemplos de llaves JSX

const nombre = 'Ane'
const horas = 30

<p>Hola, {nombre}.</p>
<p>El curso dura {horas} horas.</p>
<p>La mitad del curso son {horas / 2} horas.</p>

No usamos llaves para escribir texto fijo. Las usamos cuando queremos que React inserte un valor de JavaScript.

JSX Resultado visual
<p>Contacto</p> Muestra el texto fijo "Contacto".
<p>{tituloCurso}</p> Muestra el valor guardado en la constante.

9.5 Componentes dentro de JSX

Un componente puede usar otros componentes como si fueran etiquetas. 

<Navegacion />
<AppRoutes />
<Texto />
<Boton />

La barra final indica que el componente no tiene contenido interno. Es equivalente a abrirlo y cerrarlo sin nada dentro. 

// Forma autocerrada:
<Navegacion />

// Forma equivalente, pero sin contenido:
<Navegacion></Navegacion>

En clase usamos la forma autocerrada porque es más breve y habitual cuando el componente no recibe contenido entre apertura y cierre.

En la página de inicio usamos esta idea para introducir componentes pequeños. Texto muestra un párrafo y Boton muestra un botón visual. Todavía no tienen props ni eventos porque esta clase se centra en estructura. 

<Texto />
<Boton />

9.6 Props básicas

Las props son valores que pasamos a un componente o etiqueta JSX. En esta clase aparecen props como to, path y element. 

<Link to="/contacto">Contacto</Link>
<Route path="/contacto" element={<Contacto />} />
Prop Dónde aparece Qué significa
to Link URL interna a la que queremos navegar.
path Route URL que React Router debe reconocer.
element Route Componente que se renderiza cuando coincide la ruta.
className Etiquetas JSX Clase CSS. En JSX se usa className, no class.

Cómo leer una prop

<Link to="/contacto">Contacto</Link>

Lectura: "crea un enlace interno que navega a /contacto y muestra el texto Contacto". 

<Route path="/contacto" element={<Contacto />} />

Lectura: "si la URL es /contacto, renderiza el componente Contacto". La prop element usa llaves porque le estamos pasando JSX, no un texto normal.

9.7 Fragmentos

Un componente debe devolver un único bloque padre. Si queremos devolver dos secciones sin crear un div extra, usamos un fragmento. 

return (
  <>
    <section>...</section>
    <section>...</section>
  </>
)

9.8 Acceso al DOM

En main.jsx aparece una instrucción JavaScript clásica: 

const contenedor = document.getElementById('root')

Significa: busca en el documento HTML el elemento que tenga id="root". Después React monta la aplicación ahí.

Este es uno de los pocos puntos donde tocamos directamente el DOM. A partir de ahí, React se encarga de renderizar la interfaz. Por eso en los componentes no escribimos normalmente document.getElementById.

9.9 Llamadas encadenadas

createRoot(contenedor).render(
  <StrictMode>
    <App />
  </StrictMode>,
)

Primero se ejecuta createRoot(contenedor). Eso devuelve una raíz de React. Sobre esa raíz llamamos a .render() para mostrar App.

Podemos leerlo de izquierda a derecha: "crea una raíz de React en este contenedor y renderiza esta aplicación". Aunque parezca una línea compleja al principio, realmente son dos acciones encadenadas.

9.10 Objetos de configuración

En vite.config.js aparece un objeto JavaScript: 

export default defineConfig({
  plugins: [react()],
})

Las llaves crean un objeto. La propiedad plugins guarda un array con el plugin de React.

Objeto y array en el mismo ejemplo

{
  plugins: [react()],
}

El objeto tiene una propiedad llamada plugins. El valor de esa propiedad es un array. Dentro del array llamamos a react(), que activa el plugin.

Resumen JS/JSX

En esta clase no usamos todavía estado, eventos ni formularios. Pero sí usamos conceptos esenciales: módulos, constantes, funciones, JSX, props, fragmentos, objetos y arrays. Es una buena base para pasar después a listas, eventos y estado.
